\# 项目经验总结：Vivado DDS 波形发生器仿真



这个项目实现了一个基于 DDS 方法的数字波形发生器，支持正弦波、方波和三角波输出。项目使用 Vivado 2024.2 进行行为级仿真，目标器件为 Artix-7 `xc7a35tcsg324-1`，对应 EGO1 开发板常用芯片。



\---



\## 1. 项目目标



本项目的目标不是单纯写出一个波形输出模块，而是完成一个符合数字逻辑综合设计要求的模块化系统：



\- 使用 DDS 方法生成波形

\- 支持正弦波、方波、三角波

\- 使用按键模拟启动、暂停、波形切换

\- 主控部分包含不少于 3 个状态的状态机

\- 顶层模块只负责模块实例化和连线

\- 每个功能模块单独写在不同 `.v` 文件中

\- 通过 Vivado 行为级仿真验证功能

\## 3. DDS 方法理解



DDS 可以理解成一个“数字转盘”。



系统里有一个 32 位相位累加器：



```verilog

phase\_acc <= phase\_acc + freq\_word;


每来一个时钟，相位就往前走一小步。
相位走得越快，输出波形频率越高。

本项目使用 phase_acc[31:24] 作为 8 位相位地址：

assign phase_addr = phase_acc[31:24];

正弦波通过查找表得到；方波通过相位最高位判断得到；三角波通过相位地址线性映射得到。


4. 三种波形实现方式
4.1 正弦波

正弦波使用 256 点查找表：

sin_lut_256x8 u_sin_lut_256x8(
    .addr(phase_addr),
    .data(sine_data)
);
4.2 方波

方波只需要判断相位累加器最高位：

assign square_data = (phase_acc[31] == 1'b0) ? 8'd255 : 8'd0;

所以它只会输出两个值：

8'hFF
8'h00
4.3 三角波

三角波根据相位地址前半周期递增、后半周期递减：

assign triangle_data = (phase_addr[7] == 1'b0) ?
                       {phase_addr[6:0], 1'b0} :
                       (8'd255 - {phase_addr[6:0], 1'b0});
5. 状态机设计

主控状态机包含三个状态：

状态	含义
IDLE	等待启动
RUN	正常运行，DDS 相位累加器工作
HOLD	暂停输出，相位累加器停止

状态切换通过 key_start 控制。

6. 重要调试经验
6.1 不要混用旧路径源码

一开始工程中混入了旧路径下的 Verilog 文件，导致修改后的代码没有真正参与仿真。

解决方法：

新建干净工程
所有源码放在同一个目录
Add Sources 时勾选 Copy sources into project
不要同时保留旧版 dds_core.v 和新版 dds_core_fixed.v
6.2 方波看起来像三角波，不一定是代码错

Vivado 的波形窗口如果把 dac_data[7:0] 设置为 Analog 显示，可能会把 8'hFF 和 8'h00 之间用斜线连接，看起来像三角波。

解决方法：

将 dac_data[7:0] 显示为 Hexadecimal 或 Digital
单独观察 dac_data[7]
确认方波阶段 wave_led = 3'b010
6.3 截图时要分清总览图和局部图

总览图适合证明波形切换流程。
局部图适合证明每一种波形的具体形状。

7. 最终仿真结果

仿真中 key_wave 被按下两次，系统依次切换：

正弦波 -> 方波 -> 三角波

对应 LED 指示：

wave_led = 001 -> 正弦波
wave_led = 010 -> 方波
wave_led = 100 -> 三角波

这说明 DDS 波形生成和波形切换功能均正常。
